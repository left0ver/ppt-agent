# PPT Agent 接口文档

## 1. 基本信息

- 服务名称：`PPT Agent API`
- 当前版本：`0.1.0`
- FastAPI `root_path`：`/api`
- 默认本地启动地址：`http://127.0.0.1:8000`
- 完整接口前缀：`/api`

如果服务按默认方式启动，则接口示例地址为：

```text
http://127.0.0.1:8000/api
```

## 2. 接口总览

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/create_session_id` | `GET` | 创建新的会话 ID |
| `/layout_styles` | `GET` | 获取支持的 PPT 布局风格 |
| `/upload/content_files` | `POST` | 上传 PPT 内容文件 |
| `/upload/ppt_template` | `POST` | 上传 PPT 模板文件 |
| `/chat` | `POST` | 启动或恢复 PPT 生成流程，返回 SSE 流 |

## 3. 通用约定

### 3.1 会话 ID

大部分接口依赖 `thread_id` 来标识一次 PPT 生成会话。建议先调用 `/create_session_id` 获取新的 `thread_id`，再进行后续上传和生成。

### 3.2 跨域

后端启用了宽松的 CORS 配置：

- `allow_origins=["*"]`
- `allow_methods=["*"]`
- `allow_headers=["*"]`

### 3.3 文件存储目录

默认情况下，用户数据会保存在：

```text
./user_data/{thread_id}/
```

如果配置了环境变量 `USER_DATA_ROOT_DIR`，则以该目录为准。

## 4. 详细接口说明

## 4.1 创建会话 ID

- 方法：`GET`
- 路径：`/api/create_session_id`
- 描述：生成新的会话 ID，供后续上传文件和聊天流程使用

### 请求示例

```bash
curl http://127.0.0.1:8000/api/create_session_id
```

### 成功响应

```json
{
  "thread_id": "generated-thread-id"
}
```

### 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `thread_id` | `string` | 本次 PPT 生成任务的会话 ID |

---

## 4.2 获取布局风格

- 方法：`GET`
- 路径：`/api/layout_styles`
- 描述：返回当前支持的 PPT 布局风格列表

### 请求示例

```bash
curl http://127.0.0.1:8000/api/layout_styles
```

### 成功响应

```json
{
  "layout_styles": ["top_bottom", "grid"]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `layout_styles` | `string[]` | 当前支持的布局风格枚举值 |

### 当前支持的布局风格

| 枚举值 | 说明 |
| --- | --- |
| `top_bottom` | 上下结构布局 |
| `grid` | 网格布局 |

---

## 4.3 上传 PPT 内容文件

- 方法：`POST`
- 路径：`/api/upload/content_files`
- 描述：上传用于生成 PPT 的参考内容文件
- Content-Type：`multipart/form-data`

### 表单参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `thread_id` | `string` | 是 | 会话 ID |
| `files` | `file[]` | 否 | 内容文件列表，可多文件上传 |

### 文件限制

- 支持格式：`.pdf`、`.docx`、`.markdown`、`.md`
- 单文件最大大小：`20 MB`
- 每个会话只允许上传一次内容文件
- 如果当前会话目录下已经存在内容文件，再次上传会返回错误

### 请求示例

```bash
curl -X POST http://127.0.0.1:8000/api/upload/content_files \
  -F "thread_id=test-thread-id" \
  -F "files=@/path/to/source.pdf" \
  -F "files=@/path/to/notes.md"
```

### 成功响应

```json
{
  "thread_id": "test-thread-id",
  "file_dir": "user_data/test-thread-id/context_files",
  "status": "success"
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `thread_id` | `string` | 会话 ID |
| `file_dir` | `string` | 文件保存目录 |
| `status` | `string` | 固定为 `success` |

### 常见错误

#### 1. 文件类型不支持

HTTP `400`

```json
{
  "detail": "unsupported content file: demo.txt,only support {'.pdf', '.docx', '.markdown', '.md'}"
}
```

#### 2. 文件大小超限

HTTP `400`

```json
{
  "detail": "file size exceeds limit: large.pdf, max size is 20.0 MB"
}
```

#### 3. 当前会话已上传过内容文件

HTTP `400`

```json
{
  "detail": "content files already upload for current session:test-thread-id,please create a new session"
}
```

## 4.4 上传 PPT 模板文件

- 方法：`POST`
- 路径：`/api/upload/ppt_template`
- 描述：上传用户自己的 PPT 模板文件
- Content-Type：`multipart/form-data`
- 备注：代码注释说明该接口“目前没什么用，先放着”，但接口可正常调用

### 表单参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `thread_id` | `string` | 是 | 会话 ID |
| `file` | `file` | 是 | PPT 模板文件 |

### 文件限制

- 支持格式：`.pptx`、`.ppt`

### 请求示例

```bash
curl -X POST http://127.0.0.1:8000/api/upload/ppt_template \
  -F "thread_id=test-thread-id" \
  -F "file=@/path/to/template.pptx"
```

### 成功响应

```json
{
  "thread_id": "test-thread-id",
  "template_file_path": "user_data/test-thread-id/template/template.pptx",
  "status": "success"
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `thread_id` | `string` | 会话 ID |
| `template_file_path` | `string` | 模板文件保存路径 |
| `status` | `string` | 固定为 `success` |

### 常见错误

#### 文件类型不支持

HTTP `400`

```json
{
  "detail": "unsupported ppt template file: template.pdf,only support {'.pptx', '.ppt'}"
}
```

## 4.5 启动或恢复 PPT 生成流程

- 方法：`POST`
- 路径：`/api/chat`
- 描述：启动新的 PPT 生成流程，或在中断点恢复执行
- Content-Type：`application/json`
- 响应类型：`text/event-stream`

该接口是一个 SSE 流式接口。服务端会持续返回事件，直到流程结束或再次进入人工确认中断点。

### 请求体

```json
{
  "thread_id": "string",
  "type": "start | hitl_resume | abort_resume",
  "user_input": "string | object | null"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `thread_id` | `string` | 是 | 会话 ID |
| `type` | `string` | 是 | 调用类型 |
| `user_input` | `string \| object \| null` | 否 | 不同 `type` 下含义不同 |

### `type` 取值说明

| `type` | `user_input` 类型 | 说明 |
| --- | --- | --- |
| `start` | `string` | 启动全新流程。内容通常为用户的自然语言需求 |
| `hitl_resume` | `object` | 在人工确认中断点继续执行，提交用户表单或选择结果 |
| `abort_resume` | `null` | 用户主动跳过当前中断点后继续执行 |

### 请求示例

#### 1. 启动新流程

```bash
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "thread_id": "test-thread-id",
    "type": "start",
    "user_input": "我是一个学生，我想做一个关于 DeepSeek 的汇报 PPT，排版使用上下结构"
  }'
```

#### 2. 在中断点恢复，提交 PPT 基本信息

```bash
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "thread_id": "test-thread-id",
    "type": "hitl_resume",
    "user_input": {
      "target_audience": "导师以及同学",
      "user_role": "学生",
      "layout_style": "top_bottom",
      "num_pages": 10,
      "theme": "DeepSeek R1 的介绍"
    }
  }'
```

#### 3. 在中断点恢复，提交内容来源信息

```bash
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "thread_id": "test-thread-id",
    "type": "hitl_resume",
    "user_input": {
      "have_ppt_content_files": false,
      "ppt_content_source_urls": null
    }
  }'
```

#### 4. 在中断点恢复，提交模板信息

```bash
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "thread_id": "test-thread-id",
    "type": "hitl_resume",
    "user_input": {
      "have_ppt_template": false,
      "ppt_template_path": null
    }
  }'
```

#### 5. 在中断点恢复，提交最终风格

```bash
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "thread_id": "test-thread-id",
    "type": "hitl_resume",
    "user_input": {
      "user_ppt_style": "绿色简约风"
    }
  }'
```

### SSE 事件格式

服务端返回标准 SSE 格式：

```text
event: <event_name>
data: <json>
```

当前会返回以下 4 类事件。

#### 1. `interrupts`

表示流程进入人工确认或补充输入节点。

示例：

```text
event: interrupts
data: {"type":"interrupts","data":[...]}
```

`data` 中的中断项来自 LangGraph 的 `Interrupt` 对象，实际业务载荷主要有以下几类。

单个中断项至少包含以下字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 中断节点标识 |
| `value` | `object` | 业务侧真正需要展示或提交的数据 |

下面示例中的业务对象，实际通常位于中断项的 `value` 字段中。

##### a. 编辑 PPT 基本信息

```json
{
  "id": "interrupt-id",
  "value": {
    "title": "请确认或者修改以下PPT的相关信息，确认无误后点击提交",
    "type": "edit_form",
    "payload": {
      "theme": "DeepSeek R1 的介绍",
      "target_audience": "导师以及同学",
      "num_pages": 10,
      "user_role": "学生",
      "layout_style": "top_bottom"
    }
  }
}
```

`value.payload.layout_style` 可选值：

- `top_bottom`
- `grid`

##### b. 上传内容文件或填写内容来源网址

```json
{
  "id": "interrupt-id",
  "value": {
    "title": "你可以上传PPT内容相关的文件或者网站,如果没有可以直接跳过",
    "type": "upload_ppt_content_files",
    "file_type": ["pdf", "docx", "markdown", "md"]
  }
}
```

对应的 `hitl_resume.user_input` 建议格式：

```json
{
  "have_ppt_content_files": true,
  "ppt_content_source_urls": ["https://example.com/a", "https://example.com/b"]
}
```

或

```json
{
  "have_ppt_content_files": false,
  "ppt_content_source_urls": null
}
```

##### c. 上传 PPT 模板

```json
{
  "id": "interrupt-id",
  "value": {
    "title": "你可以上传一个PPT模板文件，如果没有可以直接跳过",
    "type": "upload_ppt_template",
    "file_type": ["pptx", "ppt"]
  }
}
```

对应的 `hitl_resume.user_input` 建议格式：

```json
{
  "have_ppt_template": true,
  "ppt_template_path": "user_data/test-thread-id/template/template.pptx"
}
```

或

```json
{
  "have_ppt_template": false,
  "ppt_template_path": null
}
```

##### d. 输入最终 PPT 风格

```json
{
  "id": "interrupt-id",
  "value": {
    "title": "请输入你想要的PPT的整体风格(包含颜色+风格)，例如绿色简约风，黑色科技风等",
    "type": "text_input"
  }
}
```

对应的 `hitl_resume.user_input` 建议格式：

```json
{
  "user_ppt_style": "绿色简约风"
}
```

#### 2. `current_stage`

表示当前执行阶段的实时进度说明。

示例：

```text
event: current_stage
data: {"type":"current_stage","data":"正在生成PPT的大纲"}
```

代码中出现过的阶段文案包括：

- `正在确认PPT的具体需求`
- `用户没有上传内容文件，正在根据PPT的相关信息来搜索内容`
- `正在解析用户PPT的内容文件`
- `正在解析用户提供的内容相关的网站内容`
- `正在解析用户上传的PPT模板`
- `正在生成PPT的大纲`
- `正在根据大纲来生成PPT每一页的内容`
- `正在生成PPT的初稿`
- `正在确认需要的PPT风格`
- `正在生成最终的PPT`

#### 3. `first_draft`

表示某一页 PPT 初稿已经生成。

示例：

```text
event: first_draft
data: {"first_draft_results":[{"page_index":0,"svg_content":"<svg>...</svg>","file_path":"user_data/test-thread-id/first_draft/page_1.svg"}]}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `first_draft_results` | `array` | 本次新增的初稿结果列表，通常只有一个元素 |
| `page_index` | `number` | 页码下标，从 `0` 开始 |
| `svg_content` | `string` | 该页生成的 SVG 内容 |
| `file_path` | `string` | 该页 SVG 文件保存路径 |

#### 4. `final_ppt`

表示某一页 PPT 终稿已经生成。

示例：

```text
event: final_ppt
data: {"final_ppt_results":[{"page_index":0,"svg_content":"<svg>...</svg>","file_path":"user_data/test-thread-id/final_ppt/page_1.svg"}]}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `final_ppt_results` | `array` | 本次新增的终稿结果列表，通常只有一个元素 |
| `page_index` | `number` | 页码下标，从 `0` 开始 |
| `svg_content` | `string` | 该页最终 SVG 内容 |
| `file_path` | `string` | 该页 SVG 文件保存路径 |

### 调用流程建议

建议前端按以下顺序调用：

1. 调用 `/create_session_id` 获取 `thread_id`
2. 可选调用 `/layout_styles` 获取布局选项
3. 可选调用 `/upload/content_files` 上传参考资料
4. 可选调用 `/upload/ppt_template` 上传模板
5. 调用 `/chat` 且 `type=start` 启动流程
6. 收到 `interrupts` 事件后，根据中断类型组织 `hitl_resume` 请求继续执行
7. 持续接收 `current_stage`、`first_draft`、`final_ppt` 事件，直到流程结束

### 备注

- `abort_resume` 会调用恢复流程，但传入的 `user_input` 为 `null`
- 如果用户未提供最终风格，后端会在生成终稿时默认使用 `绿色简约风`
- `first_draft` 和 `final_ppt` 返回的是逐页增量结果，不保证一次性返回完整列表

## 5. 常见前端接入注意事项

- `/chat` 是 SSE 接口，前端不要按普通 JSON 接口处理
- `thread_id` 必须在同一次会话链路中保持一致
- 上传内容文件后，如果继续在中断点提交 `have_ppt_content_files=true`，后端会从当前会话目录读取这些文件
- 上传模板文件后，在恢复接口中应提交正确的 `ppt_template_path`
- 页码字段 `page_index` 从 `0` 开始，如果前端展示页码，通常需要加 `1`
