# ppt-agent

## 功能

### 前置功能
1. 用户可以上传模板(上传pdf或者pptx的格式，并保存到template目录下，重新命名为template.pdf/.pptx,建议pdf的格式,后台使用soffice来进行格式的转换)，然后输入文本，系统会根据模板生成PPT。

  1.1 首先询问用户的PPT的目标群体，用户的角色，PPT的目的，PPT的风格（商务风，简约风，科技风），PPT的页数等信息 （用户的角色可以存在前端本地，之后便不再询问用户的角色）

  1.2. 用户提供PPT的内容（docx、markdown、pdf等）,用户上传之后可能需要对其重命名并保存在context目录下，使用mineru或者glm-ocr进行解析，解析之后可能需要使用一个agent来对内容进行整理，系统根据内容生成PPT。（agent不再进行资料的搜索的步骤） 

  1.3. 通用模式: 用户输入文本，系统根据文本生成PPT。

2. 可以对用户的输入的文本进行改写（如果用户的输入不够详细的话）  

### PPT生成的步骤

0. 资料收集，如果用户没有提供PPT所需的内容，则需要agent进行资料的收集。
1. 根据用户输入的文本，生成PPT的结构：包含整个PPT的标题以及每一页的标题以及大纲 （PPT的大纲生成）
2. 每一页PPT的内容的生成（PPT的内容策划）
    - 用户有上传文件，则让gemini来检索用户的文件内容来生成每一页的内容
    - 如果没有，则我们可以让grok来进行网络搜索来获取相关的资料来生成每一页的内容

3. 使用内容来让gemini生成一个PPT的初稿（整体的PPT的结果，不需要复杂的效果，比较简单的初稿） （PPT设计师）
  > 如果有模板，则需要根据模板来生成初稿
  > 同样初稿也需要临时保存在本地，用户可以继续地问agent来修改初稿的内容

4. 之后再让AI根据生成的初稿来生成最后的PPT （PPT的最终设计）


## 一些其他的需求

1. 可以对某一页进行修改: 生成的每个ppt的每一页都会保存在本地作为文件（定时删除），用户可以在生成的基础上再对某一页进行修改，agent只需要对某一页进行修改即可（建议只是小修改）
2. 逐页地展示生成的PPT，并且用户可以随时取消生成来避免token的消耗  
3. 支持多种风格，例如卡片式，图文混排式，纯文本式等多种风格的PPT的生成

作为精通信息架构与 SVG 编码的专家，你的任务是将完整的文字内容转化为一张高质量、结构化、具备高级感、简洁感和专业感的 SVG 演示文稿页面。要求如下：

1. 画布

SVG viewBox 必须是 0 0 1280 720。

2. 内容页的上下分割型布局

这是一种强调**纵向信息分层**的布局方式，通过将页面划分为上下两个主要区域，突出内容的主次关系，使信息展示更加清晰、有节奏感。布局应由内容本身的需求驱动，而非僵硬套用单一模板。上半部分和下半部分可以承担不同的信息角色，例如“概述 + 细节”、“标题核心观点 + 分项说明”、“主结论 + 支撑内容”等。

- 核心原则：
  - 纵向分层：页面至少分为上下两个主要内容区块，可以是 1:1、2:1、3:2 等不同比例，具体取决于内容的重要程度。
  - 层级感：更重要、更概括性的内容应放在面积更大的区域中；次要、补充、解释性内容放在较小区域中。
  - 灵活性：上下两个区域内部仍可继续拆分为 1 个或多个卡片，以适应不同内容密度。
  - 留白：上下区域之间以及内部卡片之间保持至少 `20px` 的间距。
  - 整洁性：整体布局应规整、均衡，避免头重脚轻或信息过于拥挤。

- 布局组合示例：

  - 经典概括型 (30/70 分割): 顶部约 30% 空间放置大字号的核心观点、主标题或引言；底部约 70% 空间排布详细的要点列表、多列文本或对比数据。

  - 均衡对等型 (50/50 分割): 上下各占一半。适合“理论+实践”、“问题+解决方案”或“顶部大型流程图 + 底部步骤解析”的并列展示。

  - 视觉主导型 (70/30 分割): 顶部 70% 空间作为一个巨大的画板，用于绘制复杂的系统架构图、关系网或图表占位；底部 30% 仅保留简短有力的结论或图注说明。

  - 顶部导航/时间轴型: 顶部 20% 放置一条贯穿水平的流程步骤或时间节点，底部 80% 则根据上方的节点，通过卡片或分列的形式展开详细内容。

- 区域建议：

  - 上半部分适合放置：页面标题、副标题、核心论点、关键结论、主视觉图表、整体摘要。
  - 下半部分适合放置：详细解释、分点内容、对比信息、支撑数据、流程说明、案例补充。
  - 若内容偏简洁，可采用“上大下小”形式增强视觉聚焦。
  - 若内容逻辑偏展开式，可采用“上小下大”形式保证主体内容展示空间。

1. 视觉要求

你生成的是初稿，不需要过多地考虑颜色，只需要按上面的布局要求来设计，然后保持：

* 背景为白色
* 所有字体为黑色
* 所有图形、线条、图表也尽量使用黑白灰体系即可

4. 排版要求

* 文本不要重叠
* 文字不要超出分区或卡片边界
* 各区域之间要有明确间距
* 文本层级清晰，主标题、分标题、正文在字号上要有合理区分
* 当某一区域文字较多时，应优先通过增加内边距、分栏、分组等方式优化排版，而不是让文字溢出
* 整体页面保持简洁、规整、易读
